### 2. 业务背景与范围 📈

本项目旨在通过数字化手段重构视光客户的**全生命周期管理流程** 。目标是从目前依赖手工 Excel 表格、以单次成交为中心的操作模式，转型为基于**数据驱动**、以**客户长期价值 (LTV)** 为中心的自动化运营模式 。

重点解决流失率高、换片提醒滞后、跨科室/跨产品转介绍困难以及过程质控缺失等问题 。

### 3. 角色画像 (Personas) 👥

| 角色代码 | 角色名称 | 典型职责与痛点 |
| :--- | :--- | :--- |
| **U01** | **视光医助/客服** | 痛点:每天面对海量 Excel 表格筛选回访名单,效率低;群发模板消息客户回复率低;任务多容易遗漏。  |
| **U02** | **验光师/验配师** | 痛点:接诊时无法快速调阅历史发育曲线;想做跨科室转介绍 (如转屈光) 但担心业绩无法核算。  |
| **U03** | **视光运营经理** | 痛点:无法实时监控员工执行质量;营销规则 (如换片周期) 调整需依赖 IT 改代码;无法及时发现经营指标异常。  |
| **U04** | **管理层** | 痛点:缺乏宏观的经营健康度看板,难以把控集团整体流失与复购情况。  |
| **U05** | **框架镜销售** | 痛点:难以识别柜台前的老客户是否有“升单”潜力;转介绍给角塑科后,不清楚成交情况和奖金归属。  |

### 4. 分角色用户故事 (User Stories) 💡

#### 4.1 角色：视光医助/客服 (U01)

**核心诉求：** 摆脱手工表，任务自动化，沟通智能化 。

| 编号 | 用户故事描述 (As a... I want to... So that...) | 验收标准 (AC) |
| :--- | :--- | :--- |
| US-01-01 | 作为视光医助,我想要系统每天自动推送“今日需回访换片/复查”的任务列表,以便于我不再需要人工去筛选 Excel 表格,防止漏跟。  | 1. 每日定时生成任务 (换片、复查、流失召回)。 <br>2. 任务列表按优先级排序。  |
| US-01-02 | 作为视光医助,我想要在联系客户时,系统能根据孩子的检查数据 (如度数增长值) 自动生成个性化话术,以便于我能更有温度地沟通,提高客户回复率。  | 1. 话术包含:上次检查时间、度数/眼轴变化情况、复查建议。 <br>2. 支持一键发送至企业微信。  |
| US-01-03 | 作为视光医助,我想要在企业微信侧边栏直接看到该客户的“眼健康档案”和“待办任务”,以便于我不需要在 CRM 和聊天窗口间来回切换。  | 1. 侧边栏展示客户核心临床数据与待办事项 <br>2. 支持在侧边栏直接完成任务状态更新。  |
| US-01-04 | 作为视光医助,我想要系统在我的任务即将超时 (如生成后 24 小时未处理) 时提醒我,以便于我能及时跟进,避免被质控通报。  | 1. 任务列表高亮显示即将超时的任务 <br>2. 接收超时预警通知。  |

| 编号 | 用户故事描述 (As a... I want to... So that...) | 验收标准 (AC) |
| :--- | :--- | :--- |
| US-03-01 | 作为运营经理,我想要在后台灵活配置“换片提醒周期” (如 550 天) 和“度数预警阈值” (如半年 50 度),以便于随时调整运营策略,无需开发介入。  | 1. 提供规则配置界面。 <br>2. 修改规则后,次日任务生成按新规则执行。  |

### 5. 非功能性需求 (NFR) ⚙️

1.  **数据准确性：** 必须确保从 HIS 系统同步的临床数据（度数、眼轴）准确无误，且延迟**不超过 24 小时 $\left(T+1\right)$** 。
2.  **移动端适配：** **企业微信侧边栏及任务处理界面**需适配手机端操作，方便一线人员随时使用 。
3.  **权限控制：** 严格的数据权限隔离，框架镜销售只能看到分配给自己的客户或公海客户，不可越权查看他人私海数据 。
